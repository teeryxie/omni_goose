# 《鹅鸭杀》共同知识与初标规范

本文档记录 Qwen3-Omni 初标时默认使用的游戏共同知识，目的是让模型把“看见的事实”“听到的说法”和“推理假设”分开。

## 游戏机制

- 鹅阵营通常通过完成任务、共享观察、投票放逐鸭阵营来获胜。
- 鸭阵营通常通过击杀、破坏、隐藏行踪、误导讨论来获胜。
- 中立角色可能有独立胜利条件，不能默认属于鹅或鸭。
- 会议、尸体报告、报警、投票结果和公开发言一般属于公共信息。
- 单个 POV 中看到的玩家、路线、任务界面、局部语音、未公开发现属于私有信息。

## 同步基准

多视角视频必须以第一局正式游戏开始作为全局 0 秒。

优先使用这些信号判定 `raw_start_sec`：

- 第一局进入地图，玩家角色可操控。
- 等待房、加载、倒计时结束后出现正式局 UI。
- 多个 POV 都能对上的首个公共同步事件，例如会议、报警、投票、明显同屏相遇。

不应作为同步 0 秒的内容：

- 开头寒暄、直播准备、设置调整。
- 等待房闲聊。
- 非正式局中的菜单、加载、黑屏或转场。

## 事件标注边界

- `movement` 只用于角色在地图上移动、切换房间或改变位置。
- 任务小游戏、点击连线、鼠标拖拽、UI 面板操作应标为 `task_ui` 或 `interaction`。
- 玩家口头说“某人是鸭”属于语音声明或怀疑，不是已证实事实。
- 地点、角色、任务名不确定时，应显式标 `uncertainty`，不要硬编。
- 证据字段优先写可复核的画面或语音依据，例如“画面出现会议投票界面”“P3 在左侧走廊短暂同屏”。

## Theory-of-Mind 相关信息

- 私有信息：当前 POV 看见但未公开说出的事实。
- 公共信息：会议公开发言、投票结果、所有人都可见的系统事件。
- 信念：玩家基于发言和观察形成的怀疑或判断。
- 错误信念：玩家相信但全局事件显示可能不成立的信息。
- 不确定性：模型无法确定、玩家无法确认或证据不足的状态。
